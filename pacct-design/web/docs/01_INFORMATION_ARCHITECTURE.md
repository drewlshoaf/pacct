# LoadToad Portal --- Information Architecture

## Overview

This document defines the structural and navigational architecture of
the LoadToad portal. Desktop-first. Dark-first. Engineer-focused.

## Top-Level Navigation (Left Sidebar)

-   Dashboard
-   Organization
-   Tests
-   Plans
-   Runs
-   Analytics
-   AI Insights
-   Settings

Hybrid hierarchy: Top-level objects remain persistent. Contextual
nesting appears inside specific objects.

## Core User Modes

1.  Setup Mode (Org → Test → Plan → Smoke → Publish)
2.  Operator Mode (Live run monitoring)
3.  Analyst Mode (Cross-run analysis & comparison)

These modes must be cognitively distinct.
